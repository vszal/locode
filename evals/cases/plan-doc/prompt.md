Read SPEC.md and DESIGN.md in this directory, then write an implementation plan
to PLAN.md.

The plan must break the work into milestones, and each milestone into numbered,
individually-actionable tasks. For every task give: what to build, which file it
lives in, and how you will know it is done (the test or check that proves it).
Order the tasks so that no task depends on one that comes later, and say which
milestone is the smallest thing that could be shipped and used.

Aim for at least three milestones and at least ten tasks in total. Each task
should be small enough that one person could finish it in a sitting.

Write PLAN.md as a file. Do not write any code yet.
